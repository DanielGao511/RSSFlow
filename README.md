# RSS-AI-Flow

基于 RSSHub 和 RSS-to-Telegram-Bot 的自动化信息流方案，Bot 内部深度集成了阿里云通义千问 (Qwen) AI 总结功能。

## 核心特性

*   **AI 全量全文总结**：自动汇总周期内所有文章全文，支持单次最高 200 篇内容。
*   **智能早中晚报**：根据北京时间自动生成带日期和 Emoji 的动态标题（🌅早报/🌤️午报/🌙晚报）。
*   **视觉排版优化**：采用 Telegram 原生引用气泡排版，遵循“一句话总结 + 分类深度摘要”结构。
*   **智能置顶管理**：内置 FIFO 队列，自动维护最近 6 条总结消息的置顶状态，保持频道整洁。
*   **自动数据清理**：滚动清理 48 小时前的历史文章，适配 30GB 左右磁盘环境。
*   **权限与安全**：
    *   `/summary` 指令仅限 `MANAGER` 身份管理。
    *   后台总结任务内置管理员白名单校验，防止 API 额度滥用。
*   **轻量化部署**：针对 2C1G 内存优化，支持本地镜像构建与远程一键部署。

## 快速开始

### 1. 配置
编辑 `.env` 文件，包含以下 AI 核心变量：
*   `ALIYUN_API_KEY`: 阿里云 API 密钥。
*   `MODEL_NAME`: 推荐 `qwen3.5-flash`。
*   `MANAGER`: 您的 Telegram 用户 ID。

### 2. 部署
1. 运行 `./sync_git.sh` 同步修改。
2. 运行 `./deploy_direct.sh` 执行服务器部署。

## Telegram 指令 (仅 MANAGER)

发送 `/summary` 唤起交互式菜单：
*   **切换开关**：开启/关闭 AI 总结逻辑。
*   **切换置顶**：控制总结消息的自动置顶。
*   **设置频率**：设定总结回溯的小时数（如 8h, 12h）。
*   **设置起始推送时间**：设定每日首份简报的推送时刻（如 06:00）。
